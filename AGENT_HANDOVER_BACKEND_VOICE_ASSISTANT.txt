5) Applio CLI Befehle (MVP-relevant)
- Speech-to-Speech (bereits vorhandenes Audio -> AI Voice):
  cd /Users/fabianprimus/applio/Applio
  /Users/fabianprimus/applio/Applio/.venv/bin/python core.py infer \
    --input_path /ABS/PATH/in.wav \
    --output_path /ABS/PATH/out.wav \
    --pth_path /ABS/PATH/model.pth \
    --index_path /ABS/PATH/model.index \
    --f0_method rmvpe \
    --export_format WAV \
    --embedder_model contentvec

- TTS + Voice Conversion in einem Schritt:
  cd /Users/fabianprimus/applio/Applio
  /Users/fabianprimus/applio/Applio/.venv/bin/python core.py tts \
    --tts_file /ABS/PATH/nonexistent.txt \
    --tts_text "Hallo Welt" \
    --tts_voice de-DE-KatjaNeural \
    --tts_rate 0 \
    --output_tts_path /ABS/PATH/tts.wav \
    --output_rvc_path /ABS/PATH/rvc.wav \
    --pth_path /ABS/PATH/model.pth \
    --index_path /ABS/PATH/model.index \
    --f0_method rmvpe \
    --export_format WAV \
    --embedder_model contentvec

- Hinweis zu `--tts_file`:
  - In Applio ist `tts_file` und `tts_text` beide required.
  - Wenn `tts_file` nicht existiert, wird `tts_text` verwendet (laut tts.py).

6) Kritische Voraussetzungen fuer Voice Conversion
- Fuer echte Voice Conversion werden ein Voice-Model und Index benoetigt:
  - .pth (RVC voice model)
  - .index (Feature index)
- Diese sind NICHT automatisch als nutzbares Zielstimmenmodell im VSE-Projekt vorhanden.
- Der Agent muss einen klaren Modellpfad-Standard definieren, z. B.:
  - backend/models/rvc/<voice_id>/model.pth
  - backend/models/rvc/<voice_id>/model.index
- Wichtig fuer UI-Integration: Zielstimme muss variabel waehlbar sein.
  - UI sendet nur `voice_id` (keine freien Dateipfade).
  - Backend mappt `voice_id` auf obige Modellpfade.
  - Unbekannte `voice_id` oder fehlende Dateien -> sauberer 4xx-Fehler mit Hinweis.

8) API-Plan (MVP)
- POST /assistant/respond
  - Input: audio_file ODER text, plus voice_id
  - Optionaler Input: `tts_provider` (default aus Config), z. B. `openai|edge|xtts`
  - Ablauf:
    1. Falls Audio: STT
    2. LLM Antwort erzeugen
    3. TTS (openai/edge/xtts; neutral oder direkt Applio tts)
    4. Applio infer/tts zur Zielstimme
  - Output:
    - response_text
    - output_audio_path oder stream/file response
    - metadata (dauer, model, errors/warnings)

- Empfohlene zusaetzliche Endpunkte:
  - GET /voices
    - Liefert alle verfuegbaren `voice_id` aus backend/models/rvc
    - Optional inkl. label/sprache, falls `meta.json` pro Stimme vorhanden ist
  - POST /stt
  - POST /chat
  - POST /tts
  - POST /voice/convert

11) Implementierungsregeln fuer Applio Wrapper
- subprocess mit explizitem cwd = APPLIO_ROOT starten.
- Absolute Pfade fuer alle Input/Output Dateien erzwingen.
- Keine freien Modellpfade aus API/UI akzeptieren, nur `voice_id`.
- Vor Aufruf pruefen:
  - input existiert
  - pth existiert
  - index existiert
  - output parent dir existiert
- Nach Aufruf pruefen:
  - exit code == 0
  - output file existiert und > 0 Bytes
- Fehlerfall:
  - stderr + stdout in logs erfassen
  - klare API-Fehlermeldung mit actionable hint

13) Bekannte Risiken / Stolpersteine
- Ohne .pth + .index kein echtes Voice-Cloning moeglich.
- Erster Applio-Lauf kann lange dauern (Modelldownload).
- Große Dateien und CPU/GPU Last koennen Timeouts verursachen.
- Netzwerkzugriff auf Hugging Face muss moeglich sein (initial).
- macOS Pfad-/Quarantine-Themen bei externen Files.
- XTTS kann je nach Device/RAM/VRAM deutlich langsam sein; Warmup und Timeout-Handling einplanen.
